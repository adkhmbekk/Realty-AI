# Развёртывание и эксплуатация Realty AI

## Требования
Docker + Docker Compose. Файл `.env` рядом с `docker-compose.yml` (образец — `.env.example`).

## Переменные окружения (ключевые)
- `POSTGRES_PASSWORD` — без дефолта (контейнер не стартует без него).
- `BOT_TOKEN` — токен Telegram-бота.
- `JWT_SECRET` — генерируется автоматически в томе `secrets_data`, если не задан; в бэкапы не попадает.
- `NGROK_AUTHTOKEN`, `NGROK_DOMAIN` — туннель (постоянный домен).
- `BACKUP_INTERVAL_HOURS` (по умолч. 24), `BACKUP_KEEP` (по умолч. 7).
Полный перечень и комментарии — в `.env.example`.

## Запуск / остановка
`start.bat` / `stop.bat` / `rebuild.bat` / `logs.bat` / `url.bat` (текущий адрес туннеля). Linux/macOS — соответствующие `docker compose up -d` / `down` / `logs -f`.

## Секреты и ПДн (важно)
- В системе хранятся персональные данные (телефоны собственников и клиентов). Диск машины с прод-данными **должен быть зашифрован** (BitLocker на Windows / LUKS на Linux) — иначе кража/потеря ноутбука = утечка ПДн (находка H5).
- Дампы БД содержат ПДн → **шифровать перед выносом** (gpg/age) и хранить копию **вне** машины (offsite). Незашифрованные дампы рядом с БД не защищают от потери/кражи устройства (C1/H5).
- Секреты только в `.env`/томе секретов, никогда в git. Если секрет когда-либо попадал в историю git — ротировать (Reset/Regenerate у провайдера).
- До платных клиентов оформить юр-слой ПДн (оферта/согласие/политика удаления) — ТЗ §19.6 (требует юр-проверки).

## Бэкапы и восстановление
- Авто-бэкап: сервис `backup` каждые `BACKUP_INTERVAL_HOURS` делает `pg_dump` + копию фото, держит `BACKUP_KEEP` последних. RPO ≈ интервалу бэкапа.
- **Проверка восстановления (обязательно перед первым клиентом):**
  1. Поднять чистый том БД на тестовом стенде.
  2. `restore.bat` (или `psql < dump.sql`) применить последний дамп.
  3. Восстановить каталог фото из копии.
  4. Запустить приложение, убедиться, что объекты/фото на месте.
  5. Зафиксировать измеренное время восстановления как RTO.
- Бэкап без проверенного restore не считается бэкапом (находка M15).

## Runbook восстановления (чтобы поднял не только основатель — снижает bus-factor, M22)
1. Установить Docker + Docker Compose на новую машину.
2. Получить из менеджера секретов: `.env` (с `POSTGRES_PASSWORD`, `BOT_TOKEN`, `NGROK_*`), последний дамп БД, копию фото, том секретов (или дать сгенерироваться новому `JWT_SECRET` — тогда все текущие сессии разлогинятся, что безопасно).
3. `rebuild.bat` → дождаться healthcheck БД.
4. Применить дамп и восстановить фото (см. раздел выше).
5. `url.bat` → убедиться, что домен туннеля активен; при необходимости проверить кнопку запуска Mini App в BotFather.
6. Проверить `/health` и вход в приложение.

## Проверка здоровья
`/health` — наружу через Caddy. Должен подтверждать доступность БД и хранилища фото (см. QW6).
